### 11:11 图片点击放大 — 彻底修复（事件委托方案）

- **根因诊断**：
  1. 首次尝试：inline `onclick` + `window.openLightbox`，但 Astro `<script type="module">` 作用域隔离导致 onclick 找不到函数
  2. 二次尝试：用 `{JSON.stringify(...)}` 注入画廊数组到脚本，TypeScript 解析器把 `<string, string[]>` 当 JSX 标签报错
  3. 三次尝试：加 `type="text/javascript"` 绕过 TS，但 Astro 的 `{...}` 模板表达式也被跳过了

- **最终方案**：data 属性 + 事件委托，零模板注入
  - 画廊容器加 `data-gallery-images={JSON.stringify(section.images)}` — 在 HTML 属性中用 `&#34;` 安全编码引号，浏览器端 `JSON.parse` 解码
  - 每个卡片加 `data-lightbox-src={img}` + `data-lightbox-section={section.id}`
  - 脚本用 `closest('[data-lightbox-src]')` + `closest('.gallery-scroll')` 查找上下文
  - 所有交互（点击卡片、关闭、翻页、键盘）全走事件监听，零 `onclick` HTML 属性
  - 脚本纯 JS（无 TypeScript 标注、无箭头函数、无模板字符串），兼容 Astro hoisting + Vite 构建

- 本地构建 + CloudStudio 部署验证通过
